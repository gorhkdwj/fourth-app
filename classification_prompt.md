# 분류에 쓴 프롬프트 / Few-shot (제출용 한 컷)

피드백 한 줄씩을 아래 기준으로 **Claude가 직접** 분류한다 (긴급도 계산·날짜 변환은 코드가 담당).

## 분류 항목
- **유형** ∈ {불만, 요청, 칭찬, 문의}
- **감정** ∈ {긍정, 부정, 중립}  (별점 있으면 참고, 없으면 내용으로 추정)
- **실패차원**(불만일 때만, 그 외 `해당없음`): 신뢰성 / 응답성 / 유형성 / 제품기호

## 주 유형 규칙 (분류 흔들림 방지)
- 불만족·문제 지적이 있으면 → **불만** (개선 제안이 섞여 있어도 불만 우선)
- 단순 정보 질문 → **문의** (답하면 끝나는 것)
- 불만족 없는 순수 개선 제안 → **요청**
- 긍정 평가 → **칭찬**

## Few-shot 예시
| 내용 | 유형 | 감정 | 실패차원 |
|---|---|---|---|
| 결제는 됐는데 포인트가 안 쌓였어요. 환불해주세요. | 불만 | 부정 | 신뢰성 |
| 신메뉴 라떼 너무 달아요. 당도 조절 옵션 있으면 좋겠어요. | 불만 | 부정 | 제품기호 |
| 주문하고 20분 기다렸어요. 너무 오래 걸려요. | 불만 | 부정 | 응답성 |
| 시즌 한정 메뉴 자주 나왔으면 좋겠어요. | 요청 | 긍정 | 해당없음 |
| 케이크 퀄리티가 베이커리급이에요. 강력 추천! | 칭찬 | 긍정 | 해당없음 |
| 단체 예약도 가능한가요? 10명 정도 인원이에요. | 문의 | 중립 | 해당없음 |

> 긴급도 = 심각도(실패차원, SERVQUAL 근거) × 불만강도(별점신호·텍스트신호 중 max). 분류는 판단만, 점수는 코드가 계산.
