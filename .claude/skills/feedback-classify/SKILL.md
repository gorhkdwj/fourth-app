# 피드백 분류 Skill

카페 고객 피드백 한 줄 한 줄을 **일관된 기준**으로 분류한다. 이 Skill은 *판단*만 한다 —
날짜 변환·긴급도 계산·집계는 `src/` 파이프라인 코드가 하므로 건드리지 않는다.

## 입력
`data/feedback_raw.csv`(또는 새 피드백 CSV). 각 행: `받은날짜 · 경로 · 별점(빈칸 가능) · 내용`.

## 출력
`data/classifications.csv`에 **아직 없는 행만** 추가(append). 컬럼(정확히 이 순서):

```
받은날짜,경로,별점,내용,유형,감정,실패차원,급한이유
```

- `받은날짜·경로·별점·내용`은 **원본 값을 그대로** 복사(파이프라인이 자연키로 매칭).
- 이미 같은 피드백(받은날짜+경로+별점+내용 동일)이 있으면 **중복 추가 금지**.
- **긴급도·신호·금전손실 컬럼은 만들지 않는다** — 코드가 계산한다.

## 분류 기준

### 유형 ∈ {불만, 요청, 칭찬, 문의}
**주 유형 규칙** (혼합 표현일 때 우선순위):
- 불만족·문제 지적이 있으면 → **불만** (개선 제안이 섞여 있어도 불만 우선)
- 단순 정보 질문이면 → **문의** (답변하면 끝나는 것)
- 불만족 없는 순수 개선 제안 → **요청**
- 긍정 평가 → **칭찬**

### 감정 ∈ {긍정, 부정, 중립}
별점이 있으면 참고(1~2=부정 신호), 없으면 내용으로 추정.

### 실패차원 (유형이 '불만'일 때만, 그 외 전부 `해당없음`)
서비스 실패가 어느 영역인지. SERVQUAL 차원 기준:
- **신뢰성**: 결제·포인트·환불·주문 정확성·앱 오류 (약속한 서비스의 정확한 이행 실패)
- **응답성**: 과도한 대기·진동벨 미작동 (적시 제공 실패)
- **유형성**: 좌석·와이파이·화장실·가격 (환경·가치)
- **제품기호**: 단맛 등 단순 취향

### 급한이유
불만이면 **한 줄 근거**(예: "결제됐으나 포인트 미적립 + 환불 요구 (금전 손실)"). 불만이 아니면 빈칸.

## Few-shot (기준 고정)

| 내용 | 유형 | 감정 | 실패차원 |
|---|---|---|---|
| 결제는 됐는데 포인트가 안 쌓였어요. 환불해주세요. | 불만 | 부정 | 신뢰성 |
| 신메뉴 라떼 너무 달아요. 당도 조절 옵션 있으면 좋겠어요. | 불만 | 부정 | 제품기호 |
| 주문하고 20분 기다렸어요. 너무 오래 걸려요. | 불만 | 부정 | 응답성 |
| 시즌 한정 메뉴 자주 나왔으면 좋겠어요. | 요청 | 긍정 | 해당없음 |
| 케이크 퀄리티가 베이커리급이에요. 강력 추천! | 칭찬 | 긍정 | 해당없음 |
| 단체 예약도 가능한가요? 10명 정도 인원이에요. | 문의 | 중립 | 해당없음 |

## 절차
1. 입력 CSV의 각 행을 위 기준으로 분류.
2. `data/classifications.csv`에 신규 행만 추가(중복 제외).
3. `py src/run_pipeline.py` 실행 → `[PASS]` 확인(긴급도·검증은 코드가 처리).
4. 실패 시 메시지(예: "분류 누락 id …")를 보고 빠진 행을 채워 다시 실행.
